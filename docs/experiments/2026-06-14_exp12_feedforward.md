# Exp12 — 前馈补偿综合评估（位置模式 Stage -1）

**日期**: 2026-06-14  
**实验编号**: exp12_feedforward  
**数据目录**: `experiment_data/exp12_feedforward/`  
**CSV**: `phaseA/results.csv` (858 行), `phaseB/results.csv` (400 行), `phaseC/results.csv` (300 行)  
**运行时间**: ~30 min, 0 failed (有效 runs 1558/1900, 速度 5-6 生成失败)  

## 目的

全面评估位置模式下前馈补偿（重力+科氏力 via `mj_rne`）对击打性能的影响：
1. **Phase A 核心消融**：torque vs pos+FF vs pos-FF，验证 FF 独立贡献
2. **Phase B PD 重扫描**：FF 启用后搜索最优 Kp/Kd_ratio
3. **Phase C 噪声鲁棒**：FF 在观测噪声下是否有额外收益

## 结论

### 数据观察（Agent 生成）

1. **前馈补偿显著提升位置模式性能**：pos_ff 相比 pos_noff 命中率提升 +10.4pp（55.2% vs 44.8%），均值误差降低 23%（183.7mm vs 239.8mm）。在各球速下 pos_ff 均优于 pos_noff，尤其在 8 m/s 时差距最大（93.8% vs 77.1%，+16.7pp）。根因：FF 通过 `mj_rne` 计算重力+科氏力偏置并施加到 `qfrc_applied`，使 PD 控制器仅需处理跟踪误差，无需对抗重力 → 跟踪精度提升 → 命中率上升。

2. **位置+FF 模式全面超越力矩模式**：pos_ff 命中率比 torque 高 +18.8pp（55.2% vs 36.4%）。力矩模式虽然理论上是"全权控制"，但缺少位置约束导致挥拍轨迹更野（TCP 波动大），在高球速下容易偏离击球窗口。位置+FF 模式兼具位置约束的安全性和前馈补偿的跟踪精度，是三种模式中最优的。

3. **FF 使 TCP 速度可控**：pos_noff 的 TCP=2.50 m/s（接近安全限速 1.8 m/s 的豁免边界），而 pos_ff 的 TCP=1.78 m/s（恰好不触发限速）。FF 补偿重力后，PD 不需要产生过大的位置修正力 → 运动更平滑 → TCP 更可控。这对真机部署至关重要（真机 max_tcp_speed=1.0 m/s 更保守）。

4. **Kp=25 是 FF 下的最优刚度**：Phase B 显示 Kp=25 在所有 Kd_ratio 下均达 100% 命中，鲁棒性窗口最宽。Kp=20（exp10 无 FF 最优）在 FF 下仍有 1 个 Kd_ratio 未达 100%（Kd_r=0.10: 95%）。Kp=30 误差最低但鲁棒性反而下降（2 个 Kd_ratio 非全中）。推荐 Kp=25/Kd_r=0.08（Kd=2.0）作为新默认。

5. **Phase C 噪声无效**：三种噪声级别结果完全相同，表明 V11 的 `--obs-noise-pos/vel` 标志未实际影响 MPC 规划。可能原因：(a) 噪声注入点在 `get_ball_state()` 但 V11 主循环可能绕过了该路径；(b) KF 完美过滤了零均值噪声。需排查 V11 噪声管线。
